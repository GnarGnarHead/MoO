# Order-4 Projection Protocol

> Status: active protocol for projected objects.
>
> This note defines how MoO may talk about projected constants, circle-like
> invariants, asymptotic anchors, and other inferred objects without making them
> runtime nodes, certainties, operands, or theorems.

## Core Definition

Order 4 is not a new certainty tier.

An Order-4 projected object is:

```text
a non-operational analysis-layer anchor inferred from a named family of exact
MoO graph evidence by an explicit projection rule
```

It is:

```text
not a runtime node
not an operand
not a confirmed fact
not a theorem
not a completed object inside strict MoO
not more certain than Order 3
```

Order 4 is more abstract than Order 3, not more certain.

Governing sentence:

```text
Order-4 objects are projected inspection anchors, not certainties or operands;
they gain credibility only when they predict or organize held-out strict graph
structure better than controls.
```

The strongest evidence is not approximation accuracy. It is predictive
organization.

## Relationship To Orders 1-3

The current MoO order language remains:

```text
Order 1:
  primitive certainty, 1

Order 2:
  confirmed core-loop positive whole-number iterations

Order 3:
  constructed relational/speculative graph nodes

Order 4:
  projected limit/invariant anchors inferred from stable Order-3 families
```

Orders 1-3 describe construction or confirmation status inside the MoO graph.
Order 4 describes an analysis-layer inference from graph evidence.

Therefore:

```text
Order-4 objects cannot be operands in strict-stage MoO.
Order-4 objects cannot promote speculative nodes.
Order-4 objects cannot replace graph inspection.
Order-4 objects can guide what graph structure to inspect next.
```

## Projection Certificate

Every Order-4 proposal needs a projection certificate:

```text
name:
evidence_layer:
claim_status:
source_family:
selection_rule:
projection_rule:
primary_metric:
graph_evidence:
stability_checks:
controls:
holdout:
prediction:
kill_rule:
allowed_use:
disallowed_use:
```

Required meaning:

```text
source_family:
  exact nodes, edges, stages, corpus paths, and corpus bounds used as evidence

selection_rule:
  how the family was chosen, preferably without target leakage

projection_rule:
  deterministic recurrence, limit, invariant, product, sum, refinement, or
  compression rule mapping the source family to the projected anchor

primary_metric:
  one preregistered score, such as prediction lift, recurrence fit,
  projection stability, held-out error reduction, or organization power

graph_evidence:
  construction witnesses, neighborhoods, operation signatures, confirmation
  status, and source-corpus alignment checks

stability_checks:
  behavior across stages, bounds, or corpus variants

controls:
  matched baselines, shuffles, placebo anchors, and alternate explanations

holdout:
  strict graph data not used to define the projection

prediction:
  what graph structure the projected anchor expects to find next

kill_rule:
  exact conditions that downgrade or reject the anchor
```

If these fields cannot be filled, the result stays a speculative note or
external-probe lead.

## Evidence Standard

Positive evidence for an Order-4 object looks like:

```text
a graph-native family selected by an explicit rule
the projection remains stable across U-checkpoints and nearby bounds
the projected anchor predicts held-out nodes, edges, recurrences, or
  neighborhoods better than matched controls
the effect survives rational baselines and construction-cost controls
placebo constants or placebo invariants do not produce the same signal
```

Negative evidence includes:

```text
the projected anchor drifts strongly as U grows
many unrelated anchors fit equally well
the source family was target-selected after seeing the result
denominator, low complexity, or exposure explains the signal
graph or operation-label shuffles preserve the effect
small bound changes erase the pattern
the projection has no held-out predictive value
```

Negative evidence should downgrade the claim to `lead` or `rejected`, not be
reinterpreted poetically.

## Non-Rational Projected Constants

Use this shared category:

```text
non-rational projected constants
```

This avoids collapsing distinct classical classes:

```text
pi and e:
  transcendental

sqrt(2) and phi:
  algebraic irrational
```

Safe language:

```text
MoO can propose a non-rational projected constant as an inspection anchor from
stable rational graph families.
```

Unsafe language:

```text
MoO directly constructs pi/e/sqrt(2)/phi as completed objects.
```

## Candidate Priority

First Order-4 tests should favor anchors with strong recurrence or algebraic
signatures:

```text
stronger first targets:
  phi via Fibonacci ratios, continued fractions, and x = 1 + 1/x
  sqrt(2) via Pell-style chains, diagonals, and x*x = 2

medium target:
  e via factorial sums, (1 + 1/n)^n, and continued fractions

hardest target:
  pi via reciprocal-square mass, Euler products, polygon refinement,
  sine-zero analogies, and circle-shell completion
```

This order is methodological, not philosophical. `pi` is important, but it is
also the easiest place to smuggle in geometry or mistake approximation for
structure.

## Circle-Like Projected Objects

The current safe object is:

```text
unit_quadratic_shell
```

Safe claim:

```text
MoO can inspect rational pairs satisfying x*x + y*y = r*r.
```

An Order-4 circle-like projected invariant would require:

```text
exact rational shell family
Q-preserving transformations
refinement or completion rule
orbit or invariance behavior
relation to pi-shadow probes
controls against arbitrary quadratic or placebo curves
```

Forbidden until earned:

```text
MoO defines the Euclidean circle.
MoO has internal angle, sine, circumference, area, or real completion.
```

## Combinatorial Emergence

Internal research language may say:

```text
chaotic emergence
```

Formal reports should prefer:

```text
deterministic construction bloom
path-sensitive combinatorial emergence
stage-sensitive combinatorial emergence
neighborhood divergence under a named metric
parameter-threshold shift
```

Technical dynamical-systems language remains unearned until MoO defines:

```text
state space
evolution map
metric or topology
trajectory definition
sensitivity, recurrence, or mixing criteria
```

Do not claim:

```text
MoO is chaotic.
MoO has attractors.
MoO has bifurcations.
MoO has Lyapunov exponents.
MoO has strange attractors.
```

## Foundational Language

Safe public language:

```text
MoO may become a constructional coordinate system over existing foundations.
```

Unsafe language:

```text
MoO replaces standard foundations.
```

The stronger claim can be considered only if projected anchors organize
multiple domains by predictive graph evidence, not by philosophical coherence
alone.

## Research Artifact Rule

Every serious Order-4 experiment should produce:

```text
<experiment_id>.pre.md
<experiment_id>.json
<experiment_id>.analysis.md
```

The preregistration freezes:

```text
source-family rule
projection rule
primary metric
thresholds
controls
holdout corpus
kill rule
```

The analysis note must include failed and negative findings. A projected anchor
that cannot fail is not an Order-4 research result.

## Short Version

```text
Order 4 is a disciplined speculative instrument.
It projects inspection anchors from exact graph evidence.
It gains credibility through held-out predictive organization.
It never becomes a strict operand, certainty, or theorem by wording alone.
```
