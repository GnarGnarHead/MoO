# Branch Glossary

> Status: canonical vocabulary for MoO branch language.
>
> A branch is not a formula family or a value list. A branch is a repeated
> witnessed relation that emerges along the recurrence of `1`.

## Branch

```text
branch = same relation repeated along the unfolding of MoO
```

A branch is a structural lineage. Its members may appear at different `U`
stages, but they share a common relation-distance from certainty.

For probe work, each branch should also have a branch card: the repeated
relation, active field, witness criterion, membership reading, interaction
reading, ordinary shadow label, and weakening rule. See
`BRANCH_FORMALIZATION_NOTE.md`.

## Operator Fan Before Branch Work

MoO is not born as the positive counting spine. Its first structural event is:

```text
1 + 1 -> 2
1 - 1 -> 0
1 * 1 -> 1
1 / 1 -> 1
```

This fan gives expansion, cancellation, and preserved identity. The current
branch-lineage tooling mostly studies branches inside the positive-spine field,
but that field is one expansion-side projection of MoO, not the whole MoO
field.

## Witness

The specific emergence event that places a value into a branch reading.

```text
12 * 12 -> 144
```

and:

```text
U144 confirms 144 through the counting spine
```

are different witnesses for the same value.

## Field Of Observation

The finite saved corpus and its bounds. A relation can be permitted by MoO and
still not witnessed in a particular field of observation.

## Recurrence Frames Are Parked

Raw recurrence, the counting spine, Fibonacci/two-memory recurrence, doubling
recurrence, and triangular accumulation are recurrence frames: ways recurrence
itself may unfold or carry memory.

They are related to branches, but they are not the active branch category in
this glossary. See `RECURRENCE_FRAMES_NOTE.md`.

Frame stacking may later become a divergence study of how `1` becomes `2`, and
how `2` opens toward structures such as `3` or `4`. That is parked. Current
branch work stays inside:

```text
1, 2, 3, 4, ...
```

## Counting-Spine Reference

The current strict corpus is indexed by the counted recurrence projection:

```text
1, 2, 3, 4, ...
```

`U_n` is a slice of that recurrence frame confirmed through `n`. Branches below
are currently read inside this field unless a report says otherwise.

## Even Branch

Twofold recurrence:

```text
n -> n+n
```

The branch is the repeated self-addition relation, not merely the list
`2,4,6,8,...`.

## Self-Relation / Square Branch

Self-multiplicative recurrence:

```text
n -> n*n
```

A square value may be witnessed through the counting spine before or without
being witnessed in the saved field through `n*n`.

## Product Branch

Nontrivial multiplicative decomposition:

```text
a * b -> n
```

with `a,b > 1`.

## Product-Branch Landing

A composite value is a product-branch landing:

```text
a * b -> n
```

with `a,b > 1`.

The product relation lands on the counting-spine value.

## Prime Branch

The prime branch is a branch.

Its members share the repeated irreducibility relation along the counting
spine:

```text
n is witnessed through the counting spine,
and no nontrivial product-branch landing witnesses n
```

The irreducibility condition is how the branch is read. It is not a replacement
for the branch.

Short forms:

```text
composites are product-branch landings
primes are prime-branch members
prime-branch membership is witnessed by irreducibility condition
```

This is not just a factor label. It is a branch reading inside the counting
spine: counting continues, and the same irreducibility relation repeats where
nontrivial product decomposition does not land.

## Shell-Relation Branch

The shell-relation branch is not the formula by itself. It is the repeated
witnessed pattern where square relations and additive relation agree:

```text
x enters the self-relation branch: x*x
y enters the self-relation branch: y*y
r enters the self-relation branch: r*r
x*x and y*y enter addition
that additive result agrees with r*r
```

This is circle-adjacent only as a rational shell relation. It is not yet a
Euclidean circle.

## Interaction Branch

Where two or more relation-lineages produce, share, or illuminate the same
structure.

For geometry work, the question is:

```text
Where does the shell-relation branch interact with the square branch?
```

not:

```text
Did one retained self-product edge row exist?
```

Prime-branch structure may also participate in the interaction:

```text
Which shell-relation cases are organized by product-branch landings?
Which shell-relation cases are constrained or organized by prime-branch members?
```

## Projected Form

A subset or repeated structure of MoO whose shadow can be read as a higher
form. Examples may include `phi`, `sqrt(2)`, `pi`, or shape-like shadows when
properly supported.

Projected form is not the branch itself:

```text
square branch:
  n -> n*n

square-like projected form:
  the geometric shadow of self-relation
```

For circle language, the projected form would come after repeated shell-relation
interaction, not from the formula alone.

Projected forms are not operands, certainties, or runtime nodes.
