# Span, Triad, And Square Emergence

> Status: active conceptual note.
>
> This note records the current MoO-native placement of early shape-adjacent
> language. It keeps the structure language primary and treats geometric names
> as later shadows.

## Governing Rule

```text
MoO-native structure first.
Geometric shadow second.
```

Do not start by saying:

```text
MoO has a line.
MoO has a triangle.
MoO has a square.
```

Start with the witnessed emergence:

```text
span
adjacency
triad
closure
self-relation
branch
projected shadow
```

## Precondition: The Operator Fan

Span, triad, and square language is downstream of the first MoO asymmetry. The
system is not born as the positive spine:

```text
1 + 1 -> 2
1 - 1 -> 0
1 * 1 -> 1
1 / 1 -> 1
```

MoO reading:

```text
2:
  expansion / successor span

0:
  cancellation / first crossing

1:
  preserved identity through multiplication and division
```

The span/triad/square ladder studies the expansion side and its later
self-relation structure. It must not erase the cancellation side.

## Emergence Ladder

Current clean ladder:

```text
1:
  immediate certainty

operator fan:

1 -> 2:

1,2,3:

1*1 -> 1:

2*2 -> 4:

n*n:

square-like form:
  shadow of repeated self-relation
```

## First Span

The first span is not a Euclidean line.

It is the first witnessed separation between immediate certainty and counted
recurrence:

```text
1 + 1 -> 2
```

MoO reading:

```text
1:
  immediate certainty

2:
  counted recurrence

1 -> 2:
  first successor span / first adjacency witness
```

The line-like reading is a later shadow of repeated adjacency:

```text
1 -> 2 -> 3 -> 4 -> ...
```

with the repeated relation:

```text
n + 1 -> n+1
```

## First Triad

The first triad is not a Euclidean triangle.

It is the first three-point additive closure frame:

```text
1 + 1 -> 2
1 + 2 -> 3
3 - 2 -> 1
3 - 1 -> 2
```

MoO reading:

```text
1,2,3:
  distinct counted points held together by witnessed relations
```

The triangle-like reading is only a projected shadow if triadic closure repeats
and separates from controls. Otherwise it is just a small additive table.

Useful active term:

```text
additive triad closure
```

Parked geometric term:

```text
triangle-like shadow
```

## First Square

The square appears later than span or triad because it requires self-relation.

The degenerate case is:

```text
1 * 1 -> 1
```

This is self-relation, but it collapses back into certainty. It does not create
a new emergence.

The first nontrivial square emergence is:

```text
2 * 2 -> 4
```

MoO reading:

```text
4 through counting spine:
  value appears

2*2 -> 4:
  square branch witnesses 4

n*n:
  repeated self-relation branch

square-like form:
  geometric shadow of the stable self-relation branch
```

## Guardrails

Do not confuse these:

```text
successor path:
  1 -> 2 -> 3

additive triad:
  1,2,3 with additive and return witnesses

geometric triangle:
  requires additional structure such as incidence, non-collinearity, plane,
  side/edge distinction, area, or orientation
```

Similarly:

```text
span:
  MoO-native relation between two emerged points

line-like shadow:
  projected reading of repeated span / adjacency

Euclidean line:
  not currently defined by MoO
```

## Controls For Triad Language

The triad language is too loose if it cannot separate `1,2,3` from nearby
controls:

```text
1,2,4:
  three points, but no 1+2 -> 4 closure

2,3,4:
  adjacent points, but not first emergence and not closed by 2+3

1,3,4:
  additive closure exists, but first adjacency is skipped

2,3,5:
  additive relation exists, but belongs to a later recurrence pattern
```

Kill rule:

```text
If the witness requirement can be replaced by value equality, the result is not
MoO-native.
```
