# Documentation Index

> Status: non-destructive documentation map.
>
> This index consolidates how to read the repository without deleting, moving,
> or demoting older notes. Historical and speculative documents are preserved;
> their category tells readers how much claim weight they currently carry.

## How To Read The Repo

Use this order for a first pass:

```text
1. README.md
2. CORE_CLAIMS.md
3. PROJECT_ALIGNMENT_NOTE.md
4. EPISTEMIC_ORDER_NOTE.md
5. GRAPH_CORPUS_NOTE.md
6. RESEARCH_LENSES.md
7. ANALYSIS_TOOL_PROTOCOL.md
```

Then branch:

```text
run code / inspect corpus       -> README.md, GRAPH_CORPUS_NOTE.md
understand the theory           -> CORE_CLAIMS.md, VISION.md
interpret old reports           -> PROJECT_ALIGNMENT_NOTE.md, EXPLORATORY_LEADS.md
design a new research probe     -> RESEARCH_LENSES.md, ANALYSIS_TOOL_PROTOCOL.md
run stage-dynamics research     -> STAGE_DYNAMICS_LENS_NOTE.md, DYNAMICS_TEST_PROTOCOL.md
```

## Document Roles

Every note should be read as one of:

```text
canonical core:
  current definitions, claim boundaries, and compute rules

protocol:
  rules for testing, reporting, and preserving research

active research branch:
  live lens or strict-corpus direction that may become a stronger claim later

speculative lens:
  useful conceptual or mathematical lens, not current runtime semantics

historical exploratory archive:
  older closure-mode result or report that may contain leads but is not
  aligned strict-stage computation

working log:
  pause point, next steps, or operational note
```

When documents conflict, prefer:

```text
CORE_CLAIMS.md
PROJECT_ALIGNMENT_NOTE.md
EPISTEMIC_ORDER_NOTE.md
GRAPH_CORPUS_NOTE.md
RESEARCH_LENSES.md
```

## Preservation Rules

Do not lose information during consolidation:

```text
do not delete old notes just because their language is historical
do not rewrite exploratory claims into strict claims
do not hide speculative leads; quarantine them with layer labels
do not treat value-only reports as primary evidence
do not rename or move artifacts without updating this index
```

When adding a new note:

```text
give it a Status line
state evidence_layer and claim_status when it makes a claim
link the corpus or report artifacts it depends on
state allowed and disallowed claim language if interpretation risk is high
add it to this index
```
