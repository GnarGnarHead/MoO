# Fundamental Operator Fan

> Status: canonical correction.
>
> This note realigns MoO around the first operator event. The positive counting
> spine is a branch of MoO, not the birth of MoO and not the whole strict field.

## Governing Sentence

```text
MoO is not born as counting.
MoO is born as the operator fan of 1 against itself.
```

The first expansion is:

```text
1 + 1 -> 2
1 - 1 -> 0
1 * 1 -> 1
1 / 1 -> 1
```

This is the ignition point.

## First Asymmetry

The fundamental operators do not agree on what `1` does to itself:

```text
+ expands:
  1 + 1 -> 2

- cancels:
  1 - 1 -> 0

* preserves:
  1 * 1 -> 1

/ preserves:
  1 / 1 -> 1
```

That gives the first MoO structure:

```text
2:
  expansion

0:
  cancellation / absence / crossing

1:
  preserved identity
```

The asymmetry is not a side effect. It is the first relation field.

## Counting Is Downstream

The positive counting spine:

```text
1, 2, 3, 4, ...
```

is a branch/projection of the operator-generated field.

It must not be described as the whole MoO core.

Correct reading:

```text
positive spine:
  expansion branch / counted recurrence projection

zero:
  first cancellation point

negative values:
  native removal / opposition structure after cancellation
```

Once subtraction/removal is present, negative structure is not an optional
afterthought:

```text
0 - 1 -> -1
-1 - 1 -> -2
```

The exact signed-field generator still needs implementation discipline, but
the conceptual rule is already fixed:

```text
The strict MoO field is not positive-only.
The positive spine is one branch of the operator-generated field.
```

## Implication For Existing Corpora

The current `strict_stage_moo.py` graph corpora are useful, but they are
positive-spine strict corpora.

They should be read as:

```text
positive-spine strict corpus:
  operands are confirmed positive-spine iterations
  outputs include retained relational nodes such as zero, negatives, and
  rationals
```

They should not be read as:

```text
the complete strict MoO field
the full signed operator-fan universe
proof that zero and negatives are secondary to positives
```

This distinction preserves the value of the existing graph reports while
preventing them from overwriting the actual MoO concept.

## Relationship To Early Shadows

The previous early-emergence ladder must be read through the operator fan:

```text
1:
  immediate certainty / preserved identity

1 + 1 -> 2:
  expansion / first successor span

1 - 1 -> 0:
  cancellation / first crossing

1 * 1 -> 1:
  degenerate self-relation / identity preservation

1 / 1 -> 1:
  degenerate ratio / identity preservation
```

Then:

```text
1,2,3:
  additive triad closure inside the positive spine

2*2 -> 4:
  first nontrivial square emergence
```

So:

```text
span, triad, and square work remain useful,
but they are downstream of the first operator asymmetry.
```
